Анализирует worklog.md и создаёт DOCX документ с описанием задач, проблем и решений.---# Work Summary SkillАвтоматический генератор отчётов о проделанной работе.## Когда использоватьКогда пользователь просит:- "Создай отчёт о работе"- "Сделай саммари"- "Подведи итоги"- "Что мы сделали?"- "Сохрани отчёт"## Процесс### Шаг 1: Чтение worklogПрочитай файл `/home/z/my-project/worklog.md` если он существует. Если нет — проанализируй текущий диалог.### Шаг 2: Сбор информацииИзвлеки из worklog/dialog:- Task ID и названия задач- Какие агенты работали- Какие шаги были выполнены- Какие проблемы возникли- Как проблемы были решены- Финальный результат### Шаг 3: Генерация DOCXИспользуй docx skill для создания отчёта:```Skill(command="docx")```### Структура отчёта```markdown# Отчёт о проделанной работе## Дата: [текущая дата]## Сессия: [session_id из контекста]## Выполненные задачи- [Task ID] Название задачи  - Агент: [имя]  - Шаги: [список]  - Результат: [описание]## Проблемы и решения| Проблема | Решение ||----------|---------|| ... | ... |## Результаты- Созданные файлы- Изменённые конфигурации- Установленные компоненты## Рекомендации- Что можно улучшить- Что доделать```### Шаг 4: СохранениеСохрани в `/home/z/my-project/download/Work_Summary_[дата].docx`Формат даты: YYYY-MM-DD_HH-MM## Пример использования```Пользователь: "Создай отчёт о работе"→ Читаем worklog.md→ Парсим задачи→ Генерируем DOCX через docx skill→ Сохраняем в download/→ Сообщаем путь к файлу```## Важно- Всегда включай дату и время в название файла- Если worklog.md пуст — анализируй диалог- Добавляй технические детали (пути, команды, конфиги)- Структурируй по задачам (Task ID)
